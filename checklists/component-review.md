# Checklist: Component Review

Проверка компонентной спецификации перед принятием в `platform/components/`.

- [ ] Назначение компонента сформулировано чётко.
- [ ] API платформенно-независим (не привязан к конкретному фреймворку).
- [ ] Все состояния компонента описаны.
- [ ] Поведение при взаимодействии описано однозначно.
- [ ] Композиция (используемые токены, вложенные компоненты) указана.
- [ ] Нет дублирования с существующим компонентом из `platform/components/`.
- [ ] Терминология соответствует [constitution/glossary.md](../constitution/glossary.md).
- [ ] Если изменение ломает совместимость — оформлено как RFC с классификацией major.

## Кто проводит проверку

Lead Designer / Design Systems Architect, при поддержке Skill `design-systems-architect`.

## При несоответствии

Спецификация возвращается автору с конкретными замечаниями по невыполненным пунктам.
