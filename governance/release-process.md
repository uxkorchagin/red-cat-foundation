# Release Process

Как выпускаются новые версии Red Cat Foundation.

## Когда происходит релиз

Релиз фиксирует накопленные и прошедшие ревью изменения (см. [review-process.md](review-process.md)) как новую версию Foundation. Релиз не привязан к календарному графику — он происходит по решению Lead Designer / Design Systems Architect, когда накоплен значимый набор изменений или требуется выпустить отдельное breaking-изменение.

## Шаги релиза

1. Определяется версия согласно [versioning.md](versioning.md) (major/minor/patch) на основе характера накопленных изменений.
2. Готовятся материалы релиза в [releases/](../releases/): release notes, при необходимости — миграционные инструкции для major-версий.
3. Запись добавляется в [CHANGELOG.md](../CHANGELOG.md).
4. Версия публикуется как актуальное состояние Foundation.

## Коммуникация изменений

Каждый релиз должен явно описывать влияние на уже запущенные проекты — что меняется, что нужно делать командам, если они решат синхронизироваться (см. принцип независимости проектов после старта, [constitution/principles.md](../constitution/principles.md)).
