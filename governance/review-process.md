# Review Process

Как проводится архитектурное ревью изменений в Red Cat Foundation.

## Назначение ревью

Ревью существует для того, чтобы любое изменение в Foundation сохраняло архитектурную целостность системы, прежде чем стать частью источника истины.

## Что подлежит ревью

- Любое RFC-предложение, прошедшее обсуждение (см. [contribution-process.md](contribution-process.md)).
- Изменения в `platform/` (токены, компоненты).
- Изменения в `constitution/` и `governance/`.
- Универсальные улучшения, предлагаемые к возврату из проектов в Foundation (см. [constitution/vision.md](../constitution/vision.md)).

## Кто проводит ревью

Lead Designer / Design Systems Architect — для архитектурных изменений. Для изменений, не требующих RFC, ревью может проводить любой участник с достаточной компетенцией в соответствующей области.

## Критерии ревью

- Соответствие принципам Foundation ([constitution/principles.md](../constitution/principles.md)).
- Обоснованность решения.
- Отсутствие дублирования с уже существующими решениями.
- Соответствие используемым чек-листам ([checklists/](../checklists/)) — для компонентов и дизайна.
- Корректная классификация влияния на совместимость (major/minor/patch).

## Результат ревью

Изменение принимается, возвращается на доработку с конкретными замечаниями, либо отклоняется с объяснением причины.
