# Versioning

Политика версионирования Red Cat Foundation. Применяется к платформе в целом — не к отдельным проектам, построенным на её основе.

## Схема

Foundation использует семантическое версионирование (SemVer): `MAJOR.MINOR.PATCH`.

### Major

Breaking change — изменение, требующее ручной адаптации со стороны команд проектов:

- изменение архитектуры компонентов;
- удаление или переименование токенов;
- изменение API компонентов в Figma;
- изменение обязательных процессов;
- изменение структуры Foundation.

### Minor

Новые возможности, компоненты, процессы и стандарты без нарушения совместимости с уже запущенными проектами.

### Patch

Исправления ошибок, улучшения документации, оптимизация существующих решений без изменения контракта.

## Распространение изменений

Обновления Foundation не распространяются на запущенные проекты автоматически. Каждый проект — независимая реализация. После выпуска новой версии команда проекта самостоятельно решает, синхронизироваться ли с ней полностью, частично или не синхронизироваться вовсе. Каждое изменение версии сопровождается описанием влияния на существующие проекты (см. [release-process.md](release-process.md)).

## Журнал версий

История версий фиксируется в [CHANGELOG.md](../CHANGELOG.md) (краткая хронологическая запись). Детальные материалы по конкретной версии — в [releases/](../releases/).
