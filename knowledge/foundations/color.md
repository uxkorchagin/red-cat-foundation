# Обоснование: структура Color

Зафиксировано Skill `design-systems-architect` после утверждения пользователем. Итоговая структура — в [platform/tokens/color.md](../../platform/tokens/color.md).

## Исследование

Adobe Spectrum, IBM Carbon, Material 3 разделяют шкалу (палитру оттенков с равномерными шагами светлоты) и роль (функциональное назначение цвета в интерфейсе). Роль остаётся неизменной между темами и брендами; меняется только шкала, на которую она ссылается. Все изученные системы фиксируют контраст как структурное правило (WCAG 2.1: 4.5:1 для текста, 3:1 для крупного текста/UI), не зависящее от конкретных оттенков.

## Решение — таксономия ролей

**Альтернатива A — плоский список ролей.** Отклонена: не масштабируется при росте числа ролей, превращается в нечитаемый перечень без структуры.

**Альтернатива B (принята) — ролевая таксономия по категориям UI** (`background`, `content`, `border`, `interactive`, `feedback`), напрямую соответствующая конвенции именования `категория/роль/вариант(/состояние)` из Token Architecture. Масштабируется по категориям, а не перечислением.

## Уточнение: форма шкалы — тоже Foundation-инвариант

При заполнении Global-таблицы обнаружился пробел: Semantic-роли ссылались на Global, но сама форма шкалы (число шагов, группы) ранее не была разделена явно на "форму" (которую можно зафиксировать сейчас) и "значение" (бренд-конфигурация). Исправлено по аналогии с уже принятым решением для Spacing — число и имена шагов (11, от 0 до 1000) и набор групп (neutral/accent/success/warning/danger/info) зафиксированы как Foundation-инвариант; бренд-конфигурацией остаётся только сам оттенок за каждым шагом. Без этого уточнения структура Color оставалась неполной — Semantic-токены ссылались в пустоту.

## Почему это решение долговечно

Роли определены по смыслу использования, а не по конкретному цвету — добавление нового бренда или смена палитры заполняют существующие роли новыми значениями шкалы и не требуют пересмотра таксономии. Feedback-роли зафиксированы по смыслу на уровне Foundation, чтобы success/danger/warning/info означали одно и то же во всех продуктах студии, независимо от того, каким оттенком это выражено.
