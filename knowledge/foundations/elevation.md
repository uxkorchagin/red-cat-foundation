# Обоснование: структура Elevation

Зафиксировано Skill `design-systems-architect` после утверждения пользователем. Итоговая структура — в [platform/tokens/elevation.md](../../platform/tokens/elevation.md).

## Исследование

Ранние версии Material выражали высоту поверхности исключительно тенью — числовая шкала уровней, маппящаяся на параметры тени. Более поздняя практика (Material 3) показала: тень плохо читается в тёмной теме — тёмная тень на тёмном фоне не создаёт достаточного контраста для считывания сигнала высоты. Решение зрелых систем — дополнить тень тональным сдвигом цвета поверхности, который работает в обеих темах и становится основным сигналом в тёмной.

## Решение 1 — механизм выражения высоты

**Альтернатива A — только тень.** Отклонена: сигнал высоты практически исчезает в тёмной теме.

**Альтернатива B (принята) — тень + тональный сдвиг поверхности, с разным весом по Theme.** Поскольку Theme уже зафиксирован как структурная ось в Token Architecture, это не абстрактная проблема, а ровно тот случай, для которого ось и предназначена.

## Решение 2 — Semantic-роли по контексту

Не отдельная альтернатива, а следствие уже принятого правила: режим применяется только на Semantic-уровне, а Elevation обязан варьироваться по Theme. Роли определены по назначению поверхности (resting/raised/menu/modal/tooltip), не по числовому уровню — тот же паттерн, что в Color, Spacing, Radius.

## Аддитивное расширение Color

Тень потребовала собственного узкого цветового значения (`color/shadow/default`), не покрытого существующими ролями Color. Зафиксировано как minor-дополнение к уже принятой структуре Color, не пересмотр её альтернатив — отражает естественный процесс, при котором поздно спроектированная подсистема обнаруживает пробел в ранее принятой, не противоречащий ей.

## Почему это решение долговечно

Elevation — единственная подсистема, явно демонстрирующая, зачем была введена ось Theme в Token Architecture: без неё проблема "тень не работает в тёмной теме" не имела бы структурного решения, кроме точечного исключения. Роли определены по смыслу использования поверхности, не по конкретным параметрам тени — изменение параметров не требует пересмотра таксономии.
