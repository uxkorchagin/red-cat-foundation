# Knowledge / Components

Обоснования архитектурных решений по компонентам: почему выбрана та или иная модель состояний, композиции, API — и накопленный опыт студии в проектировании компонентов крупных B2B-продуктов.

В отличие от [platform/components/](../../platform/components/), где находится сама спецификация компонентов, здесь фиксируется рассуждение, стоящее за решениями.

Наполняется по мере проектирования архитектуры компонентов на Этапе 1.
