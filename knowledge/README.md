# Knowledge

Единая база знаний студии. Здесь не хранятся процессы (они в [governance/](../governance/)), не хранятся шаблоны (они в [templates/](../templates/)), не хранятся инструкции по выполнению задач (они в [skills/](../skills/)). Только знания: почему принято решение, лучшие практики, исследования, рекомендации, накопленный опыт.

Каждый подраздел отвечает за свою предметную область.

## Структура

- [foundations/](foundations/) — обоснования и практики, связанные с дизайн-токенами, сетками, spacing, breakpoints и другими фундаментальными принципами визуального языка.
- [components/](components/) — обоснования архитектурных решений по компонентам, накопленный опыт.
- [figma/](figma/) — знания об архитектуре Figma, организации библиотек, работе с переменными и компонентами в Figma.
- [governance/](governance/) — знания о том, почему приняты те или иные процессные решения (в отличие от `governance/`, где описан сам процесс).

## Запланировано (см. constitution/roadmap.md)

`ux/` и `designops/` появятся при переходе к Этапу 2 и Этапу 3 соответственно. Материалы по сеткам, spacing, breakpoints и другим фундаментальным принципам до появления объективной необходимости в отдельном домене размещаются в `foundations/`.
