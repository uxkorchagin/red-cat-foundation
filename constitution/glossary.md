# Glossary

Единый словарь терминов Red Cat Foundation. Цель — устранить расхождения в терминологии между дизайнерами, аналитиками, разработчиками и Claude Code при работе с платформой.

Это прямое следствие убеждения из [vision.md](vision.md#убеждения-на-которых-строится-foundation): знание, которое не зафиксировано общим языком, легко теряется или начинает значить разное для разных людей. Глоссарий — часть того, как Foundation фиксирует своё знание, а не подразумевает его.

| Термин | Определение |
|---|---|
| **Foundation** | Red Cat Foundation целиком — общее основание, на котором строится каждый продукт студии: единый источник истины о правильном проектном решении, объединяющий токены, архитектуру компонентов, документацию и процессы. См. [vision.md](vision.md). |
| **Platform Core** | Содержательное ядро Foundation: Design Language, Component Architecture, UX Standards — платформенно-независимый источник истины о том, каким должно быть решение. В репозитории на первом этапе представлено через `platform/`. |
| **Дизайн-токен** | Именованное атомарное значение визуального языка (цвет, отступ, типографика и т.д.), не привязанное к конкретной платформе реализации. |
| **Бренд** | Набор токенов и конфигураций, задающий визуальную идентичность конкретного продукта поверх единой архитектуры Foundation. |
| **RFC (Request for Comments)** | Формальное предложение архитектурного изменения, проходящее обсуждение перед принятием решения. См. [governance/contribution-process.md](../governance/contribution-process.md). |
| **Breaking change** | Изменение, требующее ручной адаптации со стороны команд проектов: изменение архитектуры компонентов, удаление/переименование токенов, изменение API компонентов в Figma, изменение обязательных процессов, изменение структуры Foundation. |
| **Lead Designer / Design Systems Architect** | Роль, принимающая финальные архитектурные решения по Foundation. |
| **Skill** | Специализированный исполняемый агент Claude Code с определённой областью ответственности в рамках работы над Foundation. См. [skills/](../skills/). |
| **Knowledge** | Объясняющий контент платформы — почему принято решение, лучшие практики, исследования. Не процесс и не шаблон. См. [knowledge/](../knowledge/). |
