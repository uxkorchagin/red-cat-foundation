# Radius — структурная архитектура

Структура подсистемы Radius в составе Design Language (см. [README.md](README.md)). Структурно ближе к Color, чем к Spacing: степень скругления — признак визуальной идентичности бренда, а не инвариант ритма раскладки. Форма шкалы фиксируется здесь, конкретные значения — бренд-конфигурация (ось Brand), в соответствии с [token-architecture.md](token-architecture.md).

## Global — форма шкалы

Линейная шкала шагов скругления (`radius/1`, `radius/2`, `radius/3`) — число шагов (три, по числу различных размеров, фактически используемых Semantic-ролями: sm/md/lg) и именование фиксируются на уровне Foundation, конкретные значения — бренд-конфигурация.

| Token | Type | Mode | Value |
|---|---|---|---|
| `radius/1` | Number | — | → бренд (соответствует sm) |
| `radius/2` | Number | — | → бренд (соответствует md) |
| `radius/3` | Number | — | → бренд (соответствует lg) |

**`full`** не входит в линейную прогрессию — отдельный именованный категориальный токен (полное скругление: круг/пилюля), резолвящийся в значение, гарантирующее полное скругление независимо от размера элемента, а не фиксированный шаг шкалы.

## Semantic — ролевая таксономия

Semantic-уровень обязателен: Radius варьируется по оси Brand, а режим, согласно Token Architecture, применяется только на Semantic-уровне.

Mode здесь — **Brand** (единственная ось вариативности Radius); поскольку конкретные бренды ещё не определены, Mode указан как `Brand` обобщённо, без перечисления конкретных значений.

| Token | Type | Mode | Value |
|---|---|---|---|
| `radius/component/sm` | Number | Brand | → `radius/1` |
| `radius/component/md` | Number | Brand | → `radius/2` |
| `radius/component/lg` | Number | Brand | → `radius/3` |
| `radius/container/md` | Number | Brand | → `radius/2` |
| `radius/container/lg` | Number | Brand | → `radius/3` |
| `radius/full` | Number | — | 9999 (категориальное, вне шкалы) |

## Режимы

- **Brand** — основная ось вариативности: степень скругления — часть визуальной идентичности продукта.
- **Viewport** — не требуется по умолчанию; механизм доступен для точечных исключений, если они понадобятся.
- **Theme** — не влияет.

## Связь с другими подсистемами

- **[Elevation](elevation.md)** — скруглённые поверхности часто сопровождаются тенью; `radius/container` и роли Elevation проектируются согласованно.
- **[Iconography](iconography.md)** — контейнеры иконок (аватары, значки) используют `radius/full`.
