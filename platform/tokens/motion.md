# Motion — структурная архитектура

Структура подсистемы Motion в составе Design Language (см. [README.md](README.md)). В отличие от большинства подсистем, структура Motion завязана не на "что это", а на "что это сообщает": длительность и кривая движения передают причинно-следственную связь между состояниями, а не служат декоративным эффектом, в соответствии с [token-architecture.md](token-architecture.md).

## Длительность — привязана к масштабу изменения

Длительность не выражается единой плоской шкалой — она привязана к тому, насколько велико визуальное изменение:

- `motion/duration/micro` — локальные взаимодействия (наведение, переключатель, фокус).
- `motion/duration/macro` — крупные переходы (модальные окна, смена экрана/состояния).

## Кривые движения — ролевая таксономия

- `motion/easing/standard` — большинство переходов.
- `motion/easing/decelerate` — вход элемента (ускоренное замедление — "влетает и останавливается").
- `motion/easing/accelerate` — выход элемента (ускорение — "срывается с места").
- `motion/easing/emphasized` — акцентное движение, используется ограниченно, для случаев, когда движение должно специально привлечь внимание.

## Reduced motion — обязательное правило

Любой Semantic-токен Motion обязан иметь определённое поведение при включённой настройке снижения анимации (как минимум — длительность сводится к нулю/минимуму). Это структурное требование к каждой роли длительности, а не точечная настройка отдельных компонентов — аналог обязательного правила контраста в Color.

## Режимы

- **Viewport** — `motion/duration/*` может укорачиваться на компактных экранах (более резкое, "мгновенное" по ощущению взаимодействие, типичное для touch-интерфейсов).
- **Brand** — выразительность движения (выбор более "плейфул" или более сдержанной кривой `emphasized`) может быть частью идентичности бренда, по тому же принципу, что и степень скругления в Radius; сама ролевая таксономия по бренду не меняется.
- **Theme** — не влияет.

## Граница ответственности

Принципы и роли длительности/кривых — Design Language. Конкретная хореография конкретного взаимодействия (например, как именно появляется модальное окно) — решение Component Architecture/UX Standards, использующее эти токены, а не часть структуры самого Motion.

## Значения (Foundation-инвариант)

Типичные инженерные диапазоны для локальных и крупных переходов соответственно; точная bezier-функция кривых не копируется из существующих систем, определяется отдельно при технической реализации. Обоснование — [knowledge/foundations/foundation-invariant-values.md](../../knowledge/foundations/foundation-invariant-values.md).

Mode для длительности — состояние **Reduced Motion** (аналог отдельной mode-коллекции в Figma: Default/Reduced). Кривые движения режимом не варьируются (Mode = `—`).

| Token | Type | Mode | Value |
|---|---|---|---|
| `motion/duration/micro` | Number | Default | 150 |
| `motion/duration/micro` | Number | Reduced | 0 |
| `motion/duration/macro` | Number | Default | 300 |
| `motion/duration/macro` | Number | Reduced | 0 |

| Token | Type | Mode | Value |
|---|---|---|---|
| `motion/easing/standard` | String | — | Симметричная |
| `motion/easing/decelerate` | String | — | Быстрый старт — медленный финиш |
| `motion/easing/accelerate` | String | — | Медленный старт — быстрый финиш |
| `motion/easing/emphasized` | String | — | Акцентная, используется ограниченно |

## Связь с другими подсистемами

- **[Elevation](elevation.md)** — переходы между состояниями высоты поверхности используют `motion/duration/micro` и `motion/easing/standard`.
