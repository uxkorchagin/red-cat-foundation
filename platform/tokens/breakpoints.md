# Breakpoints — структурная архитектура

Структура подсистемы Breakpoints в составе Design Language (см. [README.md](README.md)). Производная система: определяет конкретные значения оси **Viewport**, объявленной как структурная возможность в [token-architecture.md](token-architecture.md), и используется [Grid](grid.md), [Typography](typography.md), [Spacing](spacing.md), [Iconography](iconography.md).

## Модель именования — абстрактные классы, не устройства

Ступени названы по доступному пространству (`compact` / `medium` / `expanded`), а не по типу устройства (mobile/tablet/desktop). Категории устройств со временем размываются (большие телефоны, складные экраны, изменяемые окна десктопа), и имя breakpoint'а, завязанное на конкретное устройство, со временем начинает не соответствовать действительности — даже если пороговая логика остаётся верной. Mobile/tablet/desktop остаются смысловым ориентиром в документации ("сегодня `compact` обычно соответствует мобильным экранам"), но не структурным именем.

## Шкала

Три ступени: `compact`, `medium`, `expanded`. Соответствует запрошенной адаптации под desktop/tablet/mobile без избыточных дополнительных градаций. Шкала структурно расширяема — добавление промежуточной или крайней ступени (например, `large` для очень широких экранов) в будущем является additive, minor-изменением, не затрагивающим уже существующие ступени.

## Пороговые значения (px)

Пороговые значения ширины экрана, в отличие от значений Color/Typography/Radius, — **инвариант Foundation**, а не точка бренд-кастомизации: все продукты студии должны согласованно понимать, где заканчивается `compact` и начинается `medium`, иначе адаптивное поведение компонентов разойдётся между проектами.

Конкретные числа здесь не определяются — откладываются в общий пакет ещё не заполненных Foundation-инвариантов вместе с базовой единицей Spacing ([spacing.md](spacing.md)), поскольку оба значения взаимосвязаны: шаг Spacing влияет на то, какие пороговые числа и количество колонок выглядят согласованными.

## Значения (Foundation-инвариант)

Диапазоны близки к опубликованным исследованиям типичной ширины устройств (window size classes), но не скопированы напрямую — выбраны собственные близкие пороги, поскольку источник один и тот же инженерный факт (распределение реальных ширин экрана), а не творческое решение, которое имеет смысл различать между системами. Обоснование — [knowledge/foundations/foundation-invariant-values.md](../../knowledge/foundations/foundation-invariant-values.md).

Эта подсистема определяет сами значения Mode для оси Viewport — поэтому Mode и Token здесь совпадают по смыслу: каждая строка — одно значение Mode коллекции Viewport в Figma Variables.

| Token | Type | Mode | Value (min-width) |
|---|---|---|---|
| `breakpoint/min-width` | Number | Compact | 0 |
| `breakpoint/min-width` | Number | Medium | 600 |
| `breakpoint/min-width` | Number | Expanded | 1024 |

## Связь с другими подсистемами

- **[Grid](grid.md)** — число колонок и `grid/container/max-width` индексируются по этим трём ступеням.
- **[Typography](typography.md)** — `typography/heading/*` и `typography/display` варьируются по этим же ступеням.
- **[Spacing](spacing.md)** — `spacing/layout/*` и `spacing/stack/*` варьируются по этим же ступеням.
- **[Iconography](iconography.md)** — `icon/size/*` потенциально варьируется по этим же ступеням.
